Chosen item: potential_improvement_5

Exact change: Set FILLNA_VALUE from 0.0 to np.nan so the trainer no longer overwrites missing values with 0 and instead lets LightGBM use its native missing-value handling.

Rationale: The IEEE-CIS fraud dataset has heavy, informative missingness across V*, D*, M*, and identity columns; replacing NaN with 0.0 collides with the many real zeros in those columns and destroys the missingness signal LightGBM would otherwise learn an optimal split direction for. FILLNA_VALUE is a discrete/categorical choice (always-fully-applied per the rules), so it is moved straight to the comment's named target (np.nan) in one move. Expected to lift validation ROC-AUC in the better (higher) direction by recovering signal that was being clobbered.
