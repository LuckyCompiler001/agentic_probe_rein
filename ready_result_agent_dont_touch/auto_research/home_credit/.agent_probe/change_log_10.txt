========================

Chosen item: potential_improvement_2

Change: Set TEST_SIZE from 0.5 to 0.2, so the train_test_split now sends
~80% of the application rows to LightGBM training and keeps ~20% for
validation, instead of the prior 50/50 split.

Why this is expected to lift validation ROC-AUC: probe_result_9 shows the
curve plateauing in the 0.7770-0.7771 band from ~epoch 110 onward — the
model is well-tuned but data-bound at this point. With TRAIN_ROW_LIMIT
already removed and the auxiliary tables already merged, the binding
constraint on further AUC is now training-set size. Moving from 50/50 to
80/20 increases training rows from ~150k to ~245k (a ~60% lift) which
should let LightGBM learn finer interactions across the rich feature
matrix and push the plateau higher. The comment names a specific value
(0.2) so it is applied directly under the categorical/discrete-choice
rule.
