Iteration 4 change log
=======================

Chosen item: potential_improvement_4

Change: Flipped USE_MINIMAL_FEATURE_SET from True to False (line 47 of train.py).

Why this should lower validation RMSE: With USE_MINIMAL_FEATURE_SET=True, determine_feature_columns hard-restricts the model to only 5 columns (Store, DayOfWeek, Promo, Open, SchoolHoliday), discarding all calendar features (Year, Month, Day, WeekOfYear, DayOfYear, IsWeekend, Quarter), all store-level features (StoreType, Assortment, CompetitionDistance, CompetitionOpenDurationMonths), and all promo-related features (Promo2, Promo2ActiveWeeks, IsPromoMonth, PromoIntervalCount). These features are well-known strong predictors for Rossmann store sales, so making them visible to LightGBM should substantially reduce validation RMSE. Boolean toggle => applied fully in one move per the always-fully-applied rule.

No revert was performed in Step 0 (probe_result_3 tail_mean equals probe_result_2 tail_mean, so no regression).
