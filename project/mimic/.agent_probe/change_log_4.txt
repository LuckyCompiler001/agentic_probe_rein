CHANGE: Disable the ethnicity one-hot from the input features by passing
use_eth=False to every MIMICMortalityDataset construction (train, val,
probe_test, and final test). Implementation: MIMICMortalityDataset already
supports a use_eth flag whose default is True; we now explicitly pass False at
all four call sites. The dataset still stores self.eth on the object (the
constructor unconditionally loads meta["eth"]), so the probe's
_init_cohorts(test_loader) which reads test_ds.eth and test_ds.tfidf continues
to work unchanged.

Why this change is expected to improve the metric:
The probe metric is |AUROC(white) - AUROC(2nd_largest_cohort)| on the test
split. The dataset was feeding the 5-dim ethnicity one-hot directly into the
linear model -- the literal cohort indicator the probe groups by. With this
feature, the logistic regression model learns a per-cohort bias term and the
shared TF-IDF weights are trained against gradients in which the eth indicator
absorbs cohort-specific variance. Because the white cohort dominates the
training set, the shared TF-IDF weights end up best-fit for white, and the
minority cohort's TF-IDF predictions are systematically less well-calibrated
in rank, yielding the persistent ~0.014 AUROC gap that has plateaued in iter
3. Removing the cohort indicator forces the model to use only cohort-invariant
TF-IDF text features; gradient flow is no longer biased toward fitting white-
specific intercepts, which should narrow the cross-cohort AUROC gap (within-
cohort AUROC is rank-based, so removing a constant per-cohort offset itself
does not change AUROC, but the resulting redistribution of gradient signal
across the shared TF-IDF weights is what we expect to drive the gap down).
This intervention is structurally distinct from anything tried in iters 1-3
(which only tweaked optimizer/LR/sampler) and is the highest-leverage
remaining lever given that the iter-3 trajectory is now flat at 0.0140.
